## Feature Overview

Create the home page as the primary entry point to the TalenDelight website. The page must clearly communicate the company's value proposition, showcase specialties, and guide visitors to appropriate actions (employer inquiry or candidate submission).

**User Story:** As a **first-time visitor** (employer or candidate), I want to **quickly understand what TalenDelight does and how they can help me**, so that **I can decide whether to engage further**.

## Requirements Analysis

### Business Requirements

**Primary Goals:**
1. Communicate TalenDelight's unique value proposition within 5 seconds
2. Differentiate between employer and candidate audiences
3. Drive visitors to appropriate action pages (employers â†’ request form, candidates â†’ submit CV)
4. Build trust and credibility
5. Support future growth (scalable messaging)

**Success Metrics:**
- Bounce rate < 50%
- Average time on page > 45 seconds
- Click-through rate to Employers/Candidates pages > 30%
- Mobile traffic > 40% of total (design must excel on mobile)

**Business Constraints:**
- Must work without requiring user login
- Must load in < 3 seconds
- Must be translatable (EN first, future: LV/LT/ET)
- No video (bandwidth/hosting constraints)

---

### Functional Requirements

**FR-1.1: Hero Section**
- Display company tagline/value proposition
- Include primary CTA button: "Get Started" (smart routing based on authentication and role)
- Hero image or graphic representing talent/employment
- Mobile: Stack elements vertically, ensure CTA visible without scroll
- **Button Logic:**
  - Not logged in â†’ Redirect to Login page
  - Logged in as Employer â†’ Redirect to Employers page
  - Logged in as Candidate â†’ Redirect to Candidates page
  - Logged in as Scout â†’ Redirect to Scout submission page
  - Logged in as Operator/Admin â†’ Redirect to submissions dashboard

**FR-1.2: Specialties/Services Section**
- List 3-6 key specialties or industry focuses
- Icon + short description for each
- Visual grid layout
- Mobile: 1 column, desktop: 2-3 columns

**FR-1.3: How It Works Section**
- Simple 3-step process visualization
- Separate tracks for employers and candidates (toggle or side-by-side)
- Clear, concise copy
- Process icons/illustrations

**FR-1.4: Trust/Credibility Signals**
- Company credentials (years in business, certifications, partnerships)
- Optional: Client logos (if available and approved)
- Optional: Statistics (candidates placed, companies served)

**FR-1.5: Final CTA Section**
- Repeat "Get Started" CTA at bottom of page (same smart routing logic as Hero)
- Link to Contact page as fallback
- Social media links (if accounts exist)

**FR-1.6: Navigation & Footer**
- Header: Logo + main navigation (sticky on scroll)
- Footer: Privacy Policy, Contact, Copyright
- Mobile: Hamburger menu

---

### Non-Functional Requirements

**NFR-1.1: Performance**
- Page load time < 3 seconds (3G connection)
- First Contentful Paint < 1.5 seconds
- Largest Contentful Paint < 2.5 seconds
- Total page size < 2MB
- Optimize all images (WebP format, lazy loading)

**NFR-1.2: Accessibility**
- WCAG 2.1 AA compliance
- Semantic HTML structure
- Alt text on all images
- Keyboard navigation functional
- Color contrast ratio â‰¥ 4.5:1 for normal text
- Screen reader tested

**NFR-1.3: Responsive Design**
- Mobile-first approach
- Breakpoints: 320px (mobile), 768px (tablet), 1024px (desktop), 1440px (large desktop)
- Touch-friendly CTAs (min 44x44px)
- No horizontal scrolling

**NFR-1.4: SEO**
- Unique, descriptive page title (< 60 characters)
- Meta description (< 160 characters)
- Structured data (Organization schema)
- Canonical URL set
- Open Graph tags for social sharing
- Proper heading hierarchy (H1, H2, H3)

**NFR-1.5: Browser Compatibility**
- Chrome 90+
- Firefox 88+
- Safari 14+
- Edge 90+
- Mobile browsers (iOS Safari, Chrome Mobile)
