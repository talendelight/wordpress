## 1. Overview

### 1.1 Purpose
Internal portal page for Managers to oversee team operations, monitor performance, manage financials, and perform all Operator functions. The page provides a strategic dashboard with team oversight capabilities and drill-down navigation to detailed views.

### 1.2 User Role
**Manager:** Team lead who supervises Operators with comprehensive access to all operational, performance, and financial data.

### 1.3 Role Hierarchy
```
Manager âŠƒ Operator âŠƒ Scout
```

**Manager has ALL Operator capabilities plus:**
- Team Performance Overview dashboard
- Monitor all Operators (workload, performance, availability)
- Monitor all Scouts (across all Operators)
- View Performance rankings (Scouts, Operators, Employers)
- Access Finance management (rates, payments, commissions)
- **View ALL candidates and employers** (read-only submitted data + workflow fields - oversight + analytics)
- Reassign work to other Operators, or Managers
- Access advanced reports and analytics
- System-wide visibility

## 14. Implementation Notes

### 14.1 WordPress Capabilities

**Create Manager-specific capability:** `manage_finance`

**Assign to Manager role:**
- All Operator capabilities
- `manage_finance`
- `view_all_candidates`
- `view_all_employers`
- `manage_operators`
- `view_performance`
- `reassign_any`
- `override_status`

### 14.2 Finance Page Security Implementation

```php
// At top of finance page template
if ( ! current_user_can( 'manage_finance' ) ) {
    wp_die( 
        __( 'Access denied. Finance page is only accessible to Managers.', 'talendelight' ),
        __( 'Access Denied', 'talendelight' ),
        array( 'response' => 403 )
    );
}

// Log access
td_log_audit_event( array(
    'action' => 'finance_page_access',
    'user_id' => get_current_user_id(),
    'ip_address' => $_SERVER['REMOTE_ADDR'],
    'timestamp' => current_time( 'mysql' ),
) );
```

### 14.3 Data Queries

**Manager queries must exclude Operator-only restrictions:**

```php
// Operator query (restricted)
$candidates = get_posts( array(
    'post_type' => 'td_candidate',
    'meta_query' => array(
        array(
            'key' => '_assigned_operator',
            'value' => get_current_user_id(),
        ),
    ),
) );

// Manager query (unrestricted)
$candidates = get_posts( array(
    'post_type' => 'td_candidate',
    // No assignment restriction
) );
```

### 14.4 Performance Optimization

**Cache expensive calculations:**
- Team metrics (cache for 5 minutes)
- Performance rankings (cache for 1 hour)
- Financial summaries (cache for 15 minutes)
- Use WordPress Transients API
