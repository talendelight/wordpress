## 1. Overview

### 1.1 Purpose
Create a public user registration form that allows candidates, employers, scouts, and other users to register without requiring account creation or login. The form captures minimal user data, validates identity documents, and creates WordPress user accounts with "pending" status awaiting admin approval.

### 1.2 User Roles Supported
- **Candidate:** External individual submitting their own profile for job opportunities
- **Employer:** Company representative looking to hire talent
- **Scout:** Talent recruiter sourcing candidates
- **Employee:** Internal team member registration (future)

### 1.3 Implementation Approach
**Custom HTML Form + Must-Use Plugin Handler**

**Why not Forminator?**
- Forminator requires GUI-based initialization that doesn't work with programmatic form creation
- Direct database imports don't trigger internal plugin registries
- Custom HTML form provides better control, maintainability, and no plugin dependencies

**Architecture:**
- **Frontend:** Custom HTML form embedded in page content (Page ID 21)
- **Backend:** `wp-content/mu-plugins/td-registration-handler.php` processes AJAX submissions
- **Storage:** WordPress users table + user meta, files uploaded to media library
- **Status:** Users created with `pending` status for approval workflow

## 2. Business Requirements

### 2.1 Anonymous Submission
**No login required to submit registration form**
- Public access to `/register-profile/?td_user_role={role}`
- Users do NOT need existing WordPress accounts
- Form directly accessible via role selection page

### 2.2 Minimal Data Capture
**Capture essential candidate data + CV file**
- **Personal Information:** First name, last name, email, phone (middle name optional)
- **Professional Information:** At least ONE of:
  - LinkedIn profile URL
  - CV/Resume file upload (PDF, DOC, DOCX, PPTX - Max 10MB)
- **Identity Verification:** National ID documents (citizenship + residence if applicable)
- **Legal Compliance:** Privacy policy and terms acceptance

### 2.3 GDPR Consent Capture
**Explicit consent with timestamp tracking**
- **Consent Checkbox:** Required field linking to Privacy Policy and Terms & Conditions
- **Timestamp Storage:** `td_registration_date` user meta field captures submission timestamp
- **User Meta Fields:**
  - `td_registration_status` = "pending" (awaiting approval)
  - `td_registration_date` (ISO 8601 timestamp)
  - All consent captured via checkbox validation before submission

### 2.4 Email Confirmation (Planned)
**Send confirmation emails to users upon submission**
- ⏳ **Candidate Confirmation:** Acknowledge registration received, explain approval process
- ⏳ **Employer Confirmation:** Registration submitted, pending verification
- ⏳ **Scout Confirmation:** Profile created, approval timeline
- 🔜 **Implementation Status:** Email templates and notification system pending (see Section 7: Future Enhancements)

### 2.5 Unique User ID Generation
**Generate WordPress User ID upon registration**
- **User ID:** WordPress automatically assigns unique numeric `ID` upon `wp_insert_user()`
- **Username Generation:** Auto-generated from email (before @ symbol + random suffix if duplicate)
- **Tracking:** User ID stored in `wp_users` table, used for:
  - User meta associations (wp_usermeta)
  - File attachment ownership (wp_posts)
  - Approval workflow tracking
  - Email notifications (when implemented)

### 2.6 Database Storage Strategy
**Store submissions in td_user_data_change_requests table with "new" status**

**⚠️ IMPORTANT:** Registration creates change requests (approval records), NOT WordPress users directly.

- **Primary Storage:** `td_user_data_change_requests` table
  - Status progression: `'new'` → `'approved'` → `'rejected'`
  - Default on submission: `'new'`
  - Manager/Operator approval required before user account creation
  
- **Change Request Fields Captured:**
  - `request_id` (varchar 20): Auto-generated unique ID (e.g., REQ-260221-1234-A1B2)
  - `record_id` (varchar 20): NULL until approved, then assigned person/company ID
  - `user_id` (int): NULL (no WordPress user exists yet at registration)
  - `role` (varchar 50): 'candidate', 'employer', 'scout', 'employee' (without td_ prefix)
  - `request_type` (varchar 20): 'register'
  - `prefix` (varchar 10): Optional name prefix (Mr., Ms., Dr., etc.)
  - `first_name` (varchar 100): Required
  - `middle_name` (varchar 100): Optional
  - `last_name` (varchar 100): Required
  - `email` (varchar 255): Required, unique check against pending/approved requests
  - `phone` (varchar 50): Required
  - `profile_method` (varchar 20): 'linkedin' or 'cv' (based on user selection)
  - `has_linkedin` (tinyint 1): 1 if LinkedIn provided, 0 otherwise
  - `has_cv` (tinyint 1): 1 if CV uploaded, 0 otherwise
  - `linkedin_url` (text): LinkedIn profile URL (if has_linkedin = 1)
  - `cv_file_path` (varchar 500): Relative path to uploaded CV (if has_cv = 1)
  - `citizenship_id_file` (varchar 500): Relative path to citizenship ID file (required)
  - `residence_id_file` (varchar 500): Relative path to residence ID file (if different from citizenship)
  - `ids_are_same` (tinyint 1): 1 if living in citizenship country, 0 if residence ID uploaded
  - `consent` (tinyint 1): Always 1 (required for submission)
  - `captcha_passed` (tinyint 1): Always 1 (client-side validation assumed)
  - `status` (varchar 20): 'new', 'approved', 'rejected'
  - `submitted_date` (datetime): Timestamp of submission
  - `approved_date` (datetime): NULL until approved
  - `approved_by_user_id` (int): NULL until approved, then manager/operator user ID
  - `rejection_reason` (text): NULL unless rejected
  
- **File Storage:** `/wp-content/uploads/register/` directory
  - **Security:** Protected with `.htaccess` (deny from all - prevent direct browser access)
  - **File Naming Convention:** `{request_id}_{type}_{original_filename}`
    - CV: `REQ-260221-1234-A1B2_cv_john-smith-resume.pdf`
    - Citizenship ID: `REQ-260221-1234-A1B2_citizenship_passport.jpg`
    - Residence ID: `REQ-260221-1234-A1B2_residence_visa.pdf`
  - **File Paths Stored:** Relative paths (e.g., `/wp-content/uploads/register/REQ-260221-1234-A1B2_cv_resume.pdf`)
  - **Access Control:** Files accessible only to authenticated users with proper permissions (manager/operator roles)
  
- **User Account Creation:** Deferred until AFTER approval
  - Registration does NOT create WordPress users
  - Users created in separate process after manager/operator approval
  - See WORDPRESS-MVP-REQUIREMENTS.md for approval workflows
  - See WORDPRESS-BACKLOG.md → Epic WP-09.5 for external user provisioning

**Status Workflow:**
```
"new" (submission) → "approved" (manager/operator) → user account created externally
                  → "rejected" (with reason) → notification sent (no account created)
```

## 5. Form Behavior & Features

### 5.1 Submission Method
- **Form Method:** POST
- **Encoding:** multipart/form-data (supports file uploads)
- **Handler:** AJAX endpoint `wp-admin/admin-ajax.php`
- **Action Hook:** `td_process_registration` (handled by must-use plugin)

### 5.2 Form Sections & Styling
1. **Hero Section:** Navy background, 56px white heading, 20px paragraph
2. **Form Container:** White card, 800px max-width, 48px padding, box-shadow
3. **Personal Info:** 3-column grid (names), 2-column grid (email/phone)
4. **Professional Info:** Two checkboxes with conditional fields (indented 26px)
5. **Identification:** Citizenship residence checkbox (checked by default), file upload buttons (solid navy #063970, 50px border-radius, auto-width), filename display
6. **Terms & Consent:** Checkbox with inline privacy/terms links
7. **Submit Button:** Solid navy (#063970), 50px border-radius (pill shape), 16px font size, hover: #2872fa (no lift effect)
8. **Footer Trust Badges:** 4 columns with SVG icons (48px) and labels

### 5.3 JavaScript Functionality
- **URL Parameter Reading:** Captures `td_user_role` from query string
- **Conditional Field Toggling:**
  - `has_linkedin` checkbox → shows/hides `linkedin_section`, sets `linkedin_url.required`
  - `has_cv` checkbox → shows/hides `cv_section`, sets `cv_file.required`
  - `live_in_citizenship_country` checkbox → shows/hides `residence_id_section` when **unchecked** (inverse logic), sets `national_id_residence.required`
- **AJAX Form Submission:**
  - Prevents page reload on submit
  - Sends data to `/wp-admin/admin-ajax.php` via Fetch API
  - Disables submit button during processing ("Submitting...")
  - Handles JSON response from backend
- **Floating Message Display:**
  - Shows success/error message in top-right corner
  - Green background (#28A745) for success
  - Red background (#DC3545) for error
  - Smooth slide-in animation (0.3s)
  - Auto-hides after 2 seconds
  - Redirects to `/welcome/` on success after message disappears
- **Form Validation (client-side):**
  - At least one checkbox selected (LinkedIn OR CV)
  - LinkedIn URL provided if checkbox checked
  - CV file uploaded if checkbox checked
  - Privacy consent checked
  - Shows floating error messages instead of alerts
- **File Upload Display:** Shows filename after selection in gray text below button
- **Button Styling:** All buttons use solid navy (#063970) background with 50px border-radius (pill shape), hover changes to #2872fa (no lift effect) to match Help page design

### 5.4 Backend Processing (td-registration-handler.php)

**Processing Steps:**
1. **Security Verification:** Verify nonce (`td_registration_nonce`) against `td_registration_form` action
2. **Field Validation:**
   - Required fields: `first_name`, `last_name`, `email`, `phone`, `td_user_role`
   - Email format validation
   - Email uniqueness check (against existing change requests with status 'new', 'pending', 'approved')
3. **Professional Information Validation:**
   - At least one of LinkedIn OR CV required
   - If LinkedIn checkbox selected: `linkedin_url` must be provided
   - If CV checkbox selected: `cv_file` must be uploaded
4. **Consent Validation:** `consent` checkbox must be checked
5. **Identity Document Validation:**
   - `national_id_citizenship` file upload required (always)
   - `national_id_residence` file upload required if `live_in_citizenship_country` unchecked
6. **Request ID Generation:** Auto-generate unique `request_id` (format: REQ-YYMMDD-HHmm-XXXX)
7. **File Upload Processing:**
   - Create `/wp-content/uploads/register/` directory if not exists
   - Protect directory with `.htaccess` (deny from all)
   - Upload files with naming convention: `{request_id}_{type}_{original_filename}`
   - Sanitize filenames before storage
   - Move files from temp location to permanent directory
   - Store relative file paths in database
8. **Database Record Creation:**
   - Insert into `td_user_data_change_requests` table
   - Status: 'new'
   - All fields populated from form data
   - Timestamp: `submitted_date` (current MySQL datetime)
9. **Response:**
   - Success: Return JSON with success message and `request_id`
   - Error: Return JSON error with specific validation failure message

**Response Format:**
```json
// Success
{
    "success": true,
    "data": {
        "message": "Registration submitted successfully! Your request is being reviewed. You will receive an email notification once your account is processed.",
        "request_id": "REQ-260221-1430-A1B2",
        "status": "new"
    }
}

// Error
{
    "success": false,
    "data": {
        "message": "Required field missing: phone"
    }
}
```

**Error Scenarios:**
- Security verification failed (invalid/missing nonce)
- Required field missing
- Invalid email format
- Email already registered (duplicate in pending/approved requests)
- Neither LinkedIn nor CV provided
- LinkedIn URL missing when checkbox selected
- CV file missing when checkbox selected
- Consent not checked
- National ID (Citizenship) upload failed
- File upload size exceeds 10MB (handled by PHP `upload_max_filesize`)
- Database insert failure

**File Upload Handling:**
- **Directory:** `/wp-content/uploads/register/`
- **Protection:** `.htaccess` with "deny from all" (prevents direct browser access)
- **File Types Accepted:**
  - CV: `.pdf`, `.doc`, `.docx`, `.pptx`
  - National IDs: `.pdf`, `.jpg`, `.jpeg`, `.png`
- **Max File Size:** 10MB (configurable in `config/uploads.ini`)
- **Naming Convention:**
  - CV: `{request_id}_cv_{sanitized_original_filename}`
  - Citizenship ID: `{request_id}_citizenship_{sanitized_original_filename}`
  - Residence ID: `{request_id}_residence_{sanitized_original_filename}`
- **Path Storage:** Relative paths stored in database (e.g., `/wp-content/uploads/register/REQ-260221-1430-A1B2_cv_resume.pdf`)

**Upload Error Handling:**
- If CV upload fails: Error added to `$upload_errors` array, but not fatal (user may have LinkedIn instead)
- If Citizenship ID upload fails: **Fatal error** - immediate JSON error response, no database insert
- If Residence ID upload fails: Error added to `$upload_errors` array, included in success message as warning
