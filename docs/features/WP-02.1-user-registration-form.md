## 1. Overview

### 1.1 Purpose
Create a public user registration form that allows candidates, employers, scouts, and other users to register without requiring account creation or login. The form captures minimal user data, validates identity documents, and creates WordPress user accounts with "pending" status awaiting admin approval.

### 1.2 User Roles Supported
- **Candidate:** External individual submitting their own profile for job opportunities
- **Employer:** Company representative looking to hire talent
- **Scout:** Talent recruiter sourcing candidates
- **Employee:** Internal team member registration (future)

### 1.3 Implementation Approach
**Custom HTML Form + Must-Use Plugin Handler**

**Why not Forminator?**
- Forminator requires GUI-based initialization that doesn't work with programmatic form creation
- Direct database imports don't trigger internal plugin registries
- Custom HTML form provides better control, maintainability, and no plugin dependencies

**Architecture:**
- **Frontend:** Custom HTML form embedded in page content (Page ID 21)
- **Backend:** `wp-content/mu-plugins/td-registration-handler.php` processes AJAX submissions
- **Storage:** WordPress users table + user meta, files uploaded to media library
- **Status:** Users created with `pending` status for approval workflow

| Field Name | Input Type | HTML ID | Display Condition | Validation |
|------------|------------|---------|-------------------|------------|
| CV Checkbox | checkbox | has_cv | Always visible | Optional (but one of two required) |

**Validation Logic:**
```javascript
if (!has_linkedin.checked && !has_cv.checked) {
    alert('Please select at least one option: LinkedIn profile or CV upload.');
    return false;
}

if (has_linkedin.checked && !linkedin_url.value) {
    alert('Please enter your LinkedIn profile URL.');
    return false;
}

if (has_cv.checked && !cv_file.files.length) {
    alert('Please upload your CV file.');
    return false;
}
```

## 4. Form Behavior & Features

### 4.1 Submission Method
- **Form Method:** POST
- **Encoding:** multipart/form-data (supports file uploads)
- **Handler:** AJAX endpoint `wp-admin/admin-ajax.php`
- **Action Hook:** `td_process_registration` (handled by must-use plugin)

### 4.2 Form Sections & Styling
1. **Hero Section:** Navy background, 56px white heading, 20px paragraph
2. **Form Container:** White card, 800px max-width, 48px padding, box-shadow
3. **Personal Info:** 3-column grid (names), 2-column grid (email/phone)
4. **Professional Info:** Two checkboxes with conditional fields (indented 26px)
5. **Identification:** File upload buttons (navy gradient), filename display
6. **Terms & Consent:** Checkbox with inline privacy/terms links
7. **Submit Button:** Navy gradient, uppercase, letter-spacing, hover effects
8. **Footer Trust Badges:** 4 columns with SVG icons (48px) and labels

### 4.3 JavaScript Functionality
- **URL Parameter Reading:** Captures `td_user_role` from query string
- **Conditional Field Toggling:**
  - `has_linkedin` checkbox → shows/hides `linkedin_section`, sets `linkedin_url.required`
  - `has_cv` checkbox → shows/hides `cv_section`, sets `cv_file.required`
  - `live_in_citizenship_country` radio → shows/hides `residence_id_section`, sets `national_id_residence.required`
- **Form Validation (on submit):**
  - At least one checkbox selected (LinkedIn OR CV)
  - LinkedIn URL provided if checkbox checked
  - CV file uploaded if checkbox checked
  - Privacy consent checked
- **File Upload Display:** Shows filename after selection in gray text below button

### 4.4 Backend Processing (td-registration-handler.php)

**Processing Steps:**
1. Verify nonce for security
2. Sanitize all input fields
3. Validate required fields
4. Check at least one of LinkedIn/CV provided
5. Upload files to WordPress media library
6. Create WordPress user with role from `td_user_role` parameter
7. Set user meta fields:
   - `td_middle_name` (middle name)
   - `td_phone` (phone number)
   - `td_registration_status` = "pending"
   - `td_registration_date` (timestamp)
   - `td_linkedin_url` (if provided)
   - `td_lives_in_citizenship_country` ("yes" or "no")
   - `td_cv_file_id` (media attachment ID, if CV uploaded)
   - `td_national_id_citizenship_id` (media attachment ID)
   - `td_national_id_residence_id` (media attachment ID, if applicable)
8. Return JSON response (success/error)

**Response Format:**
```json
{
    "success": true,
    "message": "Registration submitted successfully. Please check your email for confirmation.",
    "user_id": 123
}
```

## 5. Technical Implementation Notes

### 5.1 File Locations
- **Page Content:** Page ID 21, post_type = "page", post_name = "register-profile"

### 5.2 Database Storage
- **Users:** `wp_users` table (username derived from email, status = "pending")

- **Custom Roles:** Uses WordPress roles (td_candidate, td_employer, td_scout, etc.)

- File type validation (whitelist: .pdf, .doc, .docx, .pptx, .jpg, .jpeg, .png)
- File size limit: 10MB (enforced at upload)
- AJAX endpoint requires valid WordPress session

## 6. Deployment & Restoration

### 6.1 Deployment Workflow
1. Edit `tmp/register-profile-custom-form.html` (working copy)
2. Deploy: `podman cp tmp/register-profile-custom-form.html wp:/tmp/ && podman exec wp php /tmp/update-register-custom-form.php`
3. Flush cache: `podman exec wp wp cache flush --allow-root --skip-plugins`
4. Backup: `Copy-Item tmp/register-profile-custom-form.html restore/pages/register-profile.html`
5. Test: Visit `/register-profile/?td_user_role=td_candidate`

### 6.2 Production Deployment
- Push `restore/pages/register-profile.html` to production
- Use `restore-page-21.php` script template to restore content
- Ensure `td-registration-handler.php` is deployed to `wp-content/mu-plugins/`
- Verify SVG assets exist in theme directory
- Test all conditional logic and file uploads
