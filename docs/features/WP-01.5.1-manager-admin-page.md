## Architecture

### Page Structure
```
/managers/admin
├── Hero Section (Navy background)
├── Quick Links Grid (9 cards in 3x3 layout)
│   ├── Row 1: User Approval Requests, Manage Candidates, Manage Employers
│   ├── Row 2: Manage Scouts, Manage Operators, Performance
│   └── Row 3: View Reports, Finance, Admin Settings
└── Footer (GDPR compliance badges)
```

### Tab Interface (User Approval Requests Section)
- **Tab 1: Submitted** - New registration requests awaiting review
- **Tab 2: Approved** - Recently approved registrations (30 days)
- **Tab 3: Rejected** - Recently rejected registrations (30 days)
- **Tab 4: All** - Complete approval history

### Technical Implementation

**MU-Plugin:** `user-requests-display.php` (v1.0.0)
- Shortcode: `[user_requests_table status="..." limit="..."]`
- Role check: Manager/Administrator only
- Tab switching via JavaScript (no page reload)
- Database: `td_user_data_change_requests` table

**Page File:** `restore/pages/manager-admin-10.html`
- Pattern: `blocksy-child/card-grid-3` (3x3 grid)
- Custom JavaScript: Tab switching with color-coded backgrounds
- Parent: Managers page (ID 19)

## User Approval Workflow

### Current State (v3.4.0)
- ✅ 4-tab UI structure complete
- ✅ Placeholder "Under Development" content
- ✅ JavaScript tab switching functional
- ✅ Role-based access control active
- ⏳ Shortcodes render empty tables (no backend logic yet)

### MVP Requirements (PENG-028, PENG-035)
1. **List View** - Display all registration requests by type (Candidate, Employer, Scout, Operator, Manager)
2. **Filters** - By role, status, date range
3. **Detail View** - View full request details
4. **Actions** - Assign, Approve, Reject with reasons
5. **Notifications** - Email triggers on status changes
6. **Audit Trail** - All actions logged with timestamps

### Approval Logic (MVP)
- **Public Users (Candidate, Employer):** Operator OR Manager can approve
- **Internal Users (Scout, Manager):** ONLY Manager can approve
- **Operator Registrations:** Operator OR Manager (no self-approval)
- **All Approvals:** Require assigned reviewer before approval
- **All Rejections:** Require reason/notes
