# tmp/ Folder - Working Directory

**Purpose**: Temporary working directory for scripts, exports, and intermediate files during development

**⚠️ Important**: This folder is NOT tracked in Git (see `.gitignore`)

## Current Contents

### 📁 Directories

**elementor-exports/**
- Elementor page exports (JSON format)
- Generated by: `infra/shared/scripts/export-elementor-pages.ps1`
- Used for: Deploying Elementor pages to production

**elementor-templates/**
- Reusable Elementor templates (if any)
- Can be imported across pages/sites

**backup/**
- Local temporary backups during development
- For production backups, use `restore/backups/` instead

### 📋 Cleanup Policy

**Clean regularly** (every 2-3 weeks or after major milestones):

1. **Delete:**
   - Old SQL dumps (use `restore/` for permanent backups)
   - One-time scripts (audit, verification, deployment)
   - JSON verification files (*-check.json, *-verify.json, *-audit.json)
   - Old HTML page exports
   - Archive files (.tar.gz, .zip)
   - CSV exports (use wp-cli live queries instead)

2. **Move to restore/:**
   - Reusable page backups → `restore/pages/`
   - Important JSON exports → `restore/pages/`
   - Asset backups → `restore/assets/`

3. **Keep:**
   - `.gitignore` (prevents tmp/ from being committed)
   - Active working files (exports for current deployment)
   - elementor-exports/ (for current release)

## Usage Examples

### Generate Elementor Exports
```powershell
# Export pages to tmp/elementor-exports/
pwsh infra/shared/scripts/export-elementor-pages.ps1
```

### Create Working Backup
```bash
# Quick local backup
podman exec -it wp-db mysqldump -u root -ppassword wordpress > tmp/local-backup-$(date +%Y%m%d).sql
```

### Test Script Development
```powershell
# Test scripts here before moving to infra/shared/scripts/
.\tmp\my-test-script.ps1
```
